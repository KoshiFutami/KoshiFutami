# commit

## 目的

未コミットの差分を確認し、**意図ごとにコミットを分割**して履歴を読みやすくする。

## 手順（エージェント向け）

1. `git status -sb` と `git diff`（必要なら `git diff --cached`）で変更範囲を把握する。
2. 1 コミットにまとめすぎない。chore / feat / fix / docs など、**種類が混ざるなら分割**する。
3. ステージ: `git add -p` またはファイル単位で意図どおりに追加する。
4. メッセージは **日本語**で書く（一行目が主題）。`feat:` / `fix:` / `chore:` / `docs:` など Conventional Commits 風の接頭辞は英語のままでもよい（例: `docs: README の経歴を更新`）。本文は必要なら箇条書きで日本語。
5. 実行: `git commit`（エディタ）または `git commit -m "..."`。

## 注意

- `.env`、個人の秘密情報、シークレットをコミットしない。
- ユーザーが明示したコミットメッセージがあればそれを優先する（英語指定ならそれに従う）。
