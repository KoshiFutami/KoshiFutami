# review

## 目的

マージ前に **内容の正しさ・体裁・保守性** を確認する。コードレビュースキルがある場合はそれに沿う。

## 手順（エージェント向け）

1. 対象: `git diff main...HEAD`、またはオープンな PR（`gh pr diff`）。
2. **正しさ**: 経歴・リンク・日付・固有名詞が意図と一致するか。
3. **体裁**: Markdown の見出し階層、Kotlin ブロックの対応、英語の統一感。
4. **リンク**: URL が有効そうか、表示テキストが適切か。
5. **スコープ**: プロフィール README にふさわしい変更量か（不要なファイル追加が無いか）。

## CLI（参考）

```bash
gh pr view
gh pr diff
git diff main...HEAD
```

`make review` 等はこのリポジトリには無い。上記を直接実行する。
